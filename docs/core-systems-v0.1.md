# Terra Chronicle 大地编年史 — 核心系统深化方案 v0.1

> 状态：首轮深化稿（2026-06-12）。配套产物：`assets/concept/` 概念图 ×4、`prototype/` 活地图可运行视觉原型。

## 0. 一句话定位

**一块会呼吸的土地**：你种下的每一粒种子都是未来卡组的一张牌，你驯养的每一只灵兽都是农场的一名工人，你邻居的每一个政策都是你头顶的天气。

---

## 1. 美术方向锚定（先于一切系统）

### 1.1 风格基调：「手绘自然主义 × 编辑级极简 UI」

- **世界层（游戏画面）**：吉卜力式的温润手绘自然 + Moebius 式的开阔构图与空气感。强调**光**作为第一表现语言——四季的本质区别不是贴图换色，而是光的色温、角度、体积感的连续变化。
- **界面层（UI/HUD）**：与世界层刻意拉开质感差——日式编辑设计（杂志排版）的留白与发丝线描边，象牙白 + 墨色 + 单一暖金强调色。UI 永远不与世界争夺注意力。
- **卡牌层（实物感）**：卡牌是「玩家亲手锻造的实物」，所以牌面必须有材质叙事——胡桃木、拉丝青铜、植物雕版纹样。卡的稀有度由**工艺**表达（镶嵌、鎏金、包浆），而不是俗气的彩色边框。

### 1.2 视觉三铁律

1. **没有硬切换**：任何场景/状态切换必须有连续的视觉介质承载（光线推移、镜头拉升、季节盘旋转、卡牌翻面）。
2. **时间永远在流动**：哪怕玩家不操作，云影、风吹麦浪、灵兽踱步、光斑游移也持续发生。静帧即失败。
3. **UI 动效物理化**：所有面板进出用弹簧曲线（spring physics）而非线性补间；数值变化必须滚动而非跳变。

## 2. 活地图：四季 / 肥力 / 天气的数值模型

### 2.1 时间结构

```
1 游戏日   = 现实 ~15 分钟（单机可暂停/快进；联机全服同步走）
1 季节     = 28 游戏日（春→夏→秋→冬）
1 年代赛季 = 3 游戏年 ≈ 现实 6-8 周
```

### 2.2 地块状态向量

每个地块（tile）维护一个连续状态向量，**所有系统都读写这一个向量**，这是「经营与战斗绑定」的数据层基础：

```
tile = {
  fertility:  0..100   // 肥力：种植消耗，休耕/灵兽粪肥/豆科作物恢复
  moisture:   0..100   // 湿度：天气 + 灌溉(水系灵兽) + 邻区政策外溢
  pestLoad:   0..100   // 虫害压力：生态链失衡时上升
  manaCharge: 0..100   // 灵脉充能：魔法科技树作物需要
  biome:      enum     // 地貌底色，年代重置时重 roll
}
```

### 2.3 天气 = 全局气候 × 局部修正

- **全局气候带**由季节脚本驱动（春雨、夏旱、秋风、冬雪的概率分布）。
- **局部修正**来自：① 玩家自己的设施（风车、蓄水池）；② **邻居外溢**（见 §5）；③ 生态事件（虫灾、兽群迁徙）。
- 天气不是抽奖，是**可被经营的资源**：水系灵兽可以把一场雨「留住」3 天（蓄水），风系灵兽可以把虫灾吹向荒野。

### 2.4 作物即材料即卡牌

每种作物有三个去向，**价格/价值在三个市场间联动**，玩家的核心策略就是分配：

```
作物 → ① 出售（春季拍卖行，全服定价）
     → ② 喂养/进化灵兽（决定进化方向的「食谱系统」）
     → ③ 工坊锻造卡牌（作物=卡牌的「材质」，肥力越高的地产出的材料锻出的卡基础数值越高）
```

> 关键设计：**卡牌数值上限 = 产地土地质量的函数**。同配方卡牌，用 90 肥力土地产的星麦锻造，比 40 肥力的多 ~18% 基础值并带一条随机词条。这让「好好种地」直接等于「卡组变强」，两套系统无缝焊死。

## 3. 灵兽系统：劳动力 × 战斗力 × 生态节点

### 3.1 一兽三役

每只灵兽同时是：**农场工人**（元素技能=农活：水系灌溉、风系驱磨、土系翻耕、火系烘干）、**卡组单位**（被锻入「灵契卡」带进战斗）、**生态链节点**（在野外是食物链的一环）。

### 3.2 进化 = 食谱 + 劳役历史

同一只灵兽的进化方向由两个可被玩家经营的变量决定：

- **食谱倾向**：喂什么作物（你的田决定它的成长方向）；
- **劳役记忆**：它干过什么活（常年灌溉的水灵偏向「丰饶系」，常随你下地下城的偏向「战争系」）。

→ 没有抽卡式随机进化，**每只满级灵兽都是玩家农场史的一份履历**。这是养成深度与情感绑定的核心。

### 3.3 生态链反馈环（防止数值蛮荒）

```
过度捕猎食虫兽 → pestLoad 全区上涨 → 农田减产 → 卡牌材料质量下降
过度种植单一作物 → 该作物市场价崩 + 地块肥力单向枯竭
野外灵兽密度过低 → 虚空潮汐强度上调（自然防线变薄）
```

生态是**软约束**：不禁止任何行为，但所有竭泽而渔都会在 1-2 个季节后以可预期的方式反噬。UI 上用「大地健康度」环形仪表持续可视化，让因果可读。

## 4. 卡牌锻造与战斗

- **锻造三要素**：配方（科技树解锁）× 材料（作物/兽材/矿物，质量继承产地）× 工艺（锻造小游戏的手感分，决定词条上限）。
- **卡组构筑约束**：30 张，灵契卡（生物）≤8，季节牌组共鸣——卡牌带有产季标记，与当前现实季节同季的卡 +10% 效果，迫使卡组随季节轮换，天梯环境每季自然洗牌。
- **战斗形式**：地下城 PvE 与竞技场 PvP 共用一套**轻量回合制+站位**规则（3×3 双方棋盘），强调读牌与资源curve，单局 8-12 分钟。

---

## 5. 大陆政治层：邻里外溢模型

全服大地图按**流域**划分（不是格子国界）。你的上游邻居才能真正影响你：

- 上游大量砍伐 → 你的 `moisture` 季节均值 -15%；
- 上游建魔法塔 → 你的 `manaCharge` 自然回复 +10%；
- 流域内集体达成「护林公约」（链上投票式的季节议案）→ 全流域秋收 +8%。

→ 外交不是聊天室，是**写进地块数值的物理事实**。AI 邻居（单机态）使用同一套政策接口，所以真人替换 AI 时世界无需任何状态迁移——这是「单机联机无缝」的设计根基。

## 6. 联机架构技术选型（前端体验优先倒推）

```
客户端  TypeScript + PixiJS v8 (WebGL2/WebGPU 渲染世界层)
        + DOM/CSS 渲染 UI 层（编辑级排版交给浏览器排版引擎，动效用 spring）
        + 着色器实现四季 LUT 调色、水面、体积光、天气粒子
状态层  本地权威的单机模拟器（纯函数 reducer，每 tick 确定性演算）
        → 联机时同一模拟器跑在房间服务器上，客户端变为预测+回滚
网络    WebSocket（Cloudflare Durable Objects / Node + uWS 二选一）
        季节级全服事件用服务端定时器广播；战斗房间独立实例
存档    单机 = IndexedDB 本地存档 + 云备份；联机 = 服务器权威
        同一份 save schema（protobuf），切换模式 = 切换谁来跑模拟器
```

> **无缝切换的本质**：单机和联机跑的是**同一个确定性模拟核心**，区别只是「权威在谁手里」。AI 邻居与真人邻居走同一政策 API。这样切换时没有任何数据形态转换，只有一次平滑的镜头拉升动画（从你的农场升空到大陆视角）作为叙事转场。
