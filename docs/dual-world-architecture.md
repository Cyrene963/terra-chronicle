# 双层世界架构 — 资源隔离与防脚本设计

> 对应实现：`src/state.js`（可运行代码 + 注释），本文是设计依据。

## 为什么必须切分两层

含资源采集的联机世界必然面对两件事：**公地悲剧**（共享资源被先到者采光）和**脚本毁生态**（24h 挂机采集摧毁经济）。解法不是反作弊军备竞赛，而是**让脚本无利可图**：

```
┌─────────────────────────┐      ┌──────────────────────────────┐
│  PrivateFarm 私有位面    │      │  PublicOverworld 公共大陆     │
│  (单机/好友可见)          │      │  (全服共享,服务器权威)         │
│                         │      │                              │
│  · 全部基础资源:          │ 摘要  │  · 零基础资源(无树可砍无矿可挖) │
│    木/石/作物/灵兽        │ ───→ │  · 只有 StrategicNode:        │
│  · 作物质量=f(地块肥力)    │ 同步  │    - combat 卡组战斗闸门      │
│  · 灵兽=劳动力+战斗单位    │      │    - riddle 随机解谜闸门      │
│  · craftCard 锻造在此发生 │ ←─── │    - ritual 多人协作闸门      │
│  · 脚本无法进入他人位面    │ 奖励  │  · 流域政策→气候外溢(写进地块) │
└─────────────────────────┘      └──────────────────────────────┘
```

- **私有位面**：星露谷式安全屋。所有可挂机的产出都锁在这里，脚本最多刷自己的农场——不影响任何其他玩家，等于没有危害。
- **公共大陆**：只刷**战略级**节点，三种闸门全部要求"理解游戏"——构筑对策卡组、解随机谜题、凑齐多元素灵兽组队。无脑寻路脚本拿不到任何东西。
- **卡牌绑定**（`card.bound=true`）：锻造产物绑定锻造者，掐断工作室"打金→倒卖"链路；交易走作物/材料市场（受产量与季节自然调节）。

## 经营→战斗的焊点（craftCard 公式）

```
qualityFactor = avg(消耗作物的产地肥力) / 100        # 0.40 ~ 0.98
statScale     = 0.82 + qualityFactor × 0.36         # 同配方强度差最多 ~35%
affixCount    = 工艺分>0.85 ? 2 : >0.5 ? 1 : 0       # 锻造手感决定词条
```

实测（`node` 单元测试通过）：92 肥力土地的星麦锻出 21/30 + 2 词条；42 肥力土地同配方只有 18/25 + 0 词条。**种好地=卡变强**，两套系统从数据层焊死。

## 单机↔联机无缝的本质

`Terra.farm` 与 `Terra.overworld` 是同一 schema 在两种权威下运行：

- 单机：本地模拟器跑全部逻辑，AI 邻居通过同一政策 API 填充流域，存档进 localStorage（已实现 `Terra.save/load`）；
- 联机：farm 仍在本地/云端私有，overworld 切到服务器权威，真人替换 AI 邻居——没有任何数据迁移，只有一次镜头拉升转场。

## 下一步接线（建议顺序）

1. 把 `tileMeta`（main.js 地块档案）并入 `Terra.farm.plots`，点击面板直接读写 store
2. 种植/收获交互：体力消耗 → 作物入库（带 originFertility）
3. 工坊 UI：调用 `Terra.craftCard`，锻造成功播放卡牌翻面动画（卡面已有概念图模板）
4. 房间服务器(uWS/Durable Objects)接管 overworld,流域政策事件广播
